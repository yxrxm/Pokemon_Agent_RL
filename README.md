# Pokemon Agent RL

강화학습을 이용해 포켓몬 골드 버전을 자동으로 플레이하는 AI 에이전트 프로젝트입니다.

PyBoy 에뮬레이터 환경에서 포켓몬 골드 버전을 실행하고, 에이전트가 게임 화면과 메모리 상태를 관찰하면서 이동, 전투, 탐험, 스토리 진행을 학습하도록 구현했습니다.

본 프로젝트는 동아리 팀 프로젝트로 진행되었으며, 최종적으로 에이전트가 스스로 탐험하고 전투를 수행하여 첫 번째 체육관 관장까지 진행하는 것을 목표로 했습니다.

---

## 프로젝트 개요

기존의 `AI Plays Pokemon` 프로젝트와 포켓몬 레드 버전 강화학습 사례를 참고하여, 포켓몬 골드 버전에 맞는 강화학습 환경을 직접 구성했습니다.

에이전트는 게임 화면과 메모리 값을 기반으로 현재 상태를 파악하고, 방향키 및 A/B 버튼을 조작하며 게임을 진행합니다.

기본 구조는 다음과 같습니다.

```text
Agent → Action → Environment
Agent ← State / Reward ← Environment
```

---

## 프로젝트 목표

이 프로젝트의 목표는 단순히 캐릭터를 움직이는 것이 아니라, 포켓몬 게임의 진행 요소를 강화학습 보상으로 설계하여 에이전트가 스토리를 따라가도록 만드는 것이었습니다.

주요 목표는 다음과 같습니다.

- 포켓몬 골드 버전에서 자유 이동 기반 탐험 구현
- 전투 시스템 포함
- 스토리 중심 진행 유도
- 새로운 지역 탐험 보상 설계
- 레벨업, 경험치, 데미지, 체력 회복 등 게임 상태 기반 보상 설계
- 벽에 부딪히거나 같은 위치를 맴도는 문제 완화
- 첫 번째 체육관 관장까지 진행하는 에이전트 구현

## 강화학습 구조

에이전트는 현재 상태를 관찰한 뒤, 가능한 행동 중 하나를 선택합니다.

가능한 행동은 다음과 같습니다.

```text
아래 이동
왼쪽 이동
오른쪽 이동
위 이동
A 버튼
B 버튼
```

학습 알고리즘은 PPO를 기반으로 구성했습니다. PPO는 정책이 한 번에 너무 크게 바뀌지 않도록 제한하면서 안정적으로 학습을 진행하는 강화학습 알고리즘입니다.

## Observation 구조

환경은 단순히 게임 화면만 보는 것이 아니라, 게임 메모리에서 읽은 상태값도 함께 사용합니다.

주요 관찰값은 다음과 같습니다.

| 항목 | 설명 |
|---|---|
| screens | 최근 게임 화면 프레임 |
| health | 현재 포켓몬 파티의 체력 비율 |
| level | 포켓몬 레벨 정보 |
| badges | 획득한 배지 정보 |
| events | 게임 이벤트 플래그 |
| map | 탐험 지도 정보 |
| recent_actions | 최근 수행한 행동 기록 |

## 보상 설계

에이전트가 게임을 의미 있게 진행하도록 여러 보상 요소를 추가했습니다.

| 보상 항목 | 설명 |
|---|---|
| 탐험 보상 | 새로운 좌표를 방문하면 보상 |
| 새 지도 보상 | 새로운 맵에 진입하면 보상 |
| 주요 거점 보상 | 마을, 도로, 체육관 등 중요한 장소 진입 시 보상 |
| 체육관 보상 | 체육관 발견 시 큰 보상 |
| 레벨 보상 | 포켓몬 레벨 상승 시 보상 |
| 경험치 보상 | 전투 후 경험치 증가 시 보상 |
| 데미지 보상 | 상대 포켓몬에게 데미지를 주면 보상 |
| 전투 보상 | 전투 진입 및 승리 시 보상 |
| 회복 보상 | 체력 회복 시 보상 |
| Stuck Penalty | 같은 위치에 오래 머무르면 패널티 |

특히 에이전트가 벽에 계속 부딪히거나 같은 위치를 반복해서 맴도는 문제가 있었기 때문에, 좌표 기반 `stuck penalty`를 추가했습니다.

## 주요 구현 내용

### 1. PyBoy 기반 게임 환경 구성

PyBoy를 사용하여 포켓몬 골드 버전을 실행하고, Python 코드에서 에뮬레이터에 직접 입력을 전달하도록 구성했습니다.

```python
self.pyboy.send_input(self.valid_actions[action])
self.pyboy.tick(...)
self.pyboy.send_input(self.release_actions[action])
```

이를 통해 에이전트가 실제 게임 화면 안에서 방향키와 A/B 버튼을 조작할 수 있습니다.

---

### 2. 게임 메모리 기반 상태 읽기

포켓몬 게임 내부 메모리 주소를 활용하여 현재 체력, 레벨, 경험치, 배지, 맵 ID, 전투 상태 등을 읽었습니다.

예를 들어 다음과 같은 정보를 사용했습니다.

```text
현재 맵 ID
플레이어 좌표
포켓몬 레벨
파티 체력
상대 포켓몬 체력
전투 여부
배지 개수
이벤트 플래그
```

화면만으로는 알기 어려운 게임 진행 상태를 메모리에서 직접 읽어 보상 설계에 활용했습니다.

---

### 3. 지도 탐험 보상

에이전트가 새로운 좌표나 새로운 맵을 방문하면 보상을 받도록 설계했습니다.

주요 지역에는 더 큰 보상을 주었습니다.

```text
29번 도로
무궁시티
30번 도로
31번 도로
도라지시티
도라지 체육관
고동마을
금빛시티
```

또한 체육관에 진입하면 큰 보상을 주어 스토리 진행 방향을 강화했습니다.

---

### 4. 전투 보상

전투 중에는 상대 포켓몬의 HP 변화를 읽어서 데미지를 주었을 때 보상을 부여했습니다.

또한 트레이너 전투와 야생 포켓몬 전투를 구분하여, 트레이너 전투에서 더 큰 보상을 받을 수 있도록 했습니다.

주요 전투 보상은 다음과 같습니다.

```text
전투 진입 보상
상대에게 준 데미지 보상
경험치 획득 보상
트레이너 격파 보상
레벨업 보상
```

---

### 5. Stuck 방지 로직

강화학습 과정에서 에이전트가 다음과 같은 문제를 보였습니다.

- 지도를 보기만 하고 움직이지 않는 현상
- 벽에 계속 부딪히는 현상
- 같은 위치를 반복해서 맴도는 현상
- 특정 지역에서 정지하는 현상

이를 해결하기 위해 같은 좌표에 오래 머무르면 패널티를 주고, 일정 횟수 이상 stuck 상태가 지속되면 무작위 행동을 섞는 방식으로 개선했습니다.

```python
if self._stuck_move_count > 30:
    self.accumulated_stuck_penalty -= 0.03
```

또한 지나치게 오래 stuck 상태가 유지되면 에피소드를 강제로 종료하도록 했습니다.

## 발생한 문제와 해결

### 문제 1. 에이전트가 지도를 보기만 하는 현상

에이전트가 실제 스토리 진행보다 지도 탐색 정보에만 반응하는 문제가 있었습니다.

**해결 방법**

- 새 지역 진입 보상 추가
- 주요 거점 milestone 보상 추가
- 체육관 진입 보상 강화
- 탐험 보상 가중치 조정

---

### 문제 2. 벽에 계속 부딪히는 문제

에이전트가 이동하지 못하는 위치에서 같은 행동을 반복하는 문제가 있었습니다.

**해결 방법**

- 좌표 기반 stuck count 추가
- 같은 좌표에 오래 머무르면 패널티 부여
- 일정 횟수 이상 stuck이면 무작위 행동 적용
- 500회 이상 stuck이면 에피소드 강제 종료

---

### 문제 3. 같은 구역을 맴도는 현상

탐험 보상 때문에 이미 방문한 지역 근처를 반복해서 움직이는 문제가 발생했습니다.

**해결 방법**

- 맵 진행도 기반 보상 가중치 추가
- 진행 단계가 후퇴하면 탐험 보상을 줄이는 cut-off rule 적용
- 새로운 맵 발견 보상 추가

## 해결하지 못한 점

프로젝트 기간 내에 완전히 해결하지 못한 부분도 있었습니다.

- 특정 상황에서 전투를 비효율적으로 반복하는 문제
- PP가 부족한 기술을 계속 선택하는 문제
- 특정 지역에서 이동 효율이 낮아지는 문제
- 장기적인 스토리 진행 안정성 부족
- 더 많은 체육관까지 안정적으로 진행하지 못한 점

이후에는 전투 상황에서 사용할 수 있는 행동을 더 세분화하고, 스토리 이벤트 기반 보상을 더 정교하게 설계하면 성능을 개선할 수 있을 것으로 보입니다.

## 실행 방법

### 1. 패키지 설치

```bash
pip install pyboy gymnasium stable-baselines3 opencv-python numpy matplotlib einops
```

### 2. 설정 파일 수정

`config.py` 또는 실행 설정 파일에서 ROM 경로, 세션 저장 경로, 학습 step 등을 설정합니다.

```python
config = {
    "gb_path": "path/to/pokemon_gold.gbc",
    "session_path": "sessions/session_1",
    "headless": True,
    "max_steps": 100000,
}
```

> ROM 파일은 저작권 문제로 저장소에 포함하지 않습니다.  
> 사용자가 직접 보유한 합법적인 ROM 파일을 사용해야 합니다.

### 3. 학습 실행

```bash
python main.py
```

### 4. TensorBoard 확인

```bash
tensorboard --logdir sessions
```

단순히 모델을 학습시키는 것보다, 어떤 행동을 좋은 행동으로 정의하고 어떤 상황에 보상을 줄 것인지 설계하는 과정이 핵심이었습니다.

최종적으로 에이전트가 포켓몬 골드 환경에서 탐험, 전투, 레벨업을 수행하며 첫 번째 체육관 관장까지 진행하는 결과를 얻었습니다.
